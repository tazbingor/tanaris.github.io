Django-Web开发05-Model数据模型

发表于 2018-04-12 | 分类于 Python

Django 模型

创建app时,Django会为你的app创建一个model,继承于django.db.models.Model

Django中的Models,即抽象化模型层.对于数据操作提供支持.简而言之,不需要写过多的SQL,就能对数据库进行操作.这是Django带来的便利

(ps:SQL也很重要)

Django模型字段

常用模型字段

  • CharField()：字符串类型字段
  • BooleanField()：布尔类型字段
  • IntegerField()：整数型类型字段
  • FloatField(): 与 python 里的 float 实例相同
  • NullBooleanField(): 可以包含空值的布尔类型，相当于设置了null=True的BooleanField
  • DateField()：日期类型字段
  • DateTimeField()：日期时间类型字段,利用datetime.datetime实例来分表表示时间和日期，类似DateField。
  • FileField()：上传文件类型字段,该字段不支持primary_key和unique参数，否则类型错误。有一个必须参数upload_to，用于保存文件的本地文件系统
  • TextField()：文本类型字段

关系型字段:

  • ForeignKey()：外键关系类型字段
  • ManyToManyField()：多对多关系类型字段
  • OneToOneField()：一对一，将字段定义在任意一端中

…….

字段属性(选项)

一些模型字段中常用或独有的属性:

  • null:boolean值，是否出现空字段, 缺省则设置为false。通常不将其用于字符型字段上，比如CharField，TextField上。字符型字段如果没有值会返回空字符串。
  • blank :boolean值，该字段是否可以为空。如果为假，则必须有值。
  • choices: 元组值，一个用来选择值的2维元组。第一个值是实际存储的值，第二个用来方便进行选择。
  • db_column: string值，指定当前列在数据库中的名字，不设置，将自动采用model字段名；
  • db_index : boolean 值，如果为True将为此字段创建索引；
  • default: 给当前字段设定的缺省值，可以是一个具体值，也可以是一个可调用的对象，如果是可调用的对象将每次产生一个新的对象；
  • editable: boolean 值，如果为假，admin模式下将不能改写。缺省为真；
  • error_messages: 字典，设置默认的出错信息，可覆盖的key 有 null, blank, invalid, invalid_choice, 和 unique。
  • help_text: admin模式下帮助文档,form widget 内显示帮助文本。
  • primary_key: 设置主键，如果没有设置django创建表时会自动加上
  • radio_admin: 用于admin 模式下将select 转换 radio 显示。只用于 ForeignKey 或者设置了
  • choicesunique: boolean值，数据是否进行唯一性验证；
  • unique_for_date: 字符串类型，值指向一个DateTimeField 或者 一个 DateField的列名称。日期唯一
  • unique_for_year:用法同上
  • verbose_name:字段描述名,更人性化的描述字段
  • validators: 有效性检查。无效则抛出 django.core.validators.ValidationError 异常。

更多详细信息都在官方文档中:https://yiyibooks.cn/xx/Django_1.11.6/index.html

ORM

Django中有一个强大的功能,即ORM(Object Relational Mapping)

Django的ORM就是创建SQL去查询和操作数据库的一种方式,代表对象关系映射

  • 对象（Object）：这部分表示哪个对象或者在哪种编程语言下使用这个框架，例如Python。
  • 关系（Relational）：这部分表示你正在使用的RDBMS数据库（关系数据库管理器系统）。有许多流行的关系数据库，你可能使用以下这些：MSSQL、MySQL、Oracle、PostgreSQL、MariaDB、PerconaDB、TokuDB。关系数据库中最常见的是它们的关系结构（表，列，键，约束等）。
  • 映射（Mapping）：最后一部分表示前面两个部分之间的桥接和连接，即对象和数据库表。

ORM的作用是将编程语言与数据库进行关联，以简化依赖于数据的应用程序的创建过程
